# LEVIO — GAP9 Embedded Implementation

Optimized C implementation of the LEVIO VIO pipeline targeting the [GAP9](https://github.com/GreenWaves-Technologies/) multi-core RISC-V SoC by GreenWaves Technologies. Runs the full visual-inertial odometry pipeline in real-time using under 110 KB of L1 working memory.

## Prerequisites

- **GAP SDK** — The GAP9 SDK is proprietary and was provided by [GreenWaves Technologies](https://github.com/GreenWaves-Technologies/). Contact the current IP holder for access. Source the environment so that `GAP_SDK_HOME` is set.
- **CMake** >= 3.19
- **Sensor data** — The pipeline reads grayscale camera frames and IMU measurements, both as PGM files. IMU data is encoded as 28x10 PGM images (10 measurements per frame, serialized as raw bytes), matching the format an IMU sensor would provide in batches synchronized with the camera. Generate the data from any EuRoC rosbag sequence using the provided conversion script (see [Data Preparation](#data-preparation)). The resulting directory structure should be:

```
sensor_data/demo/
├── images/
│   ├── frame000000.pgm      160x120 grayscale, rectified and subsampled
│   ├── frame000001.pgm
│   └── ...
├── imu/
│   ├── imu000001.pgm         28x10 raw bytes (10 IMU samples per frame)
│   ├── imu000002.pgm
│   └── ...
└── start_stop_indices.h      Frame range (auto-generated by script)
```

## Data Preparation

A helper script converts any EuRoC MAV rosbag into the PGM format expected by the pipeline. It undistorts and rectifies the camera images using the EuRoC cam0 intrinsics, subsamples them to 160x120, and batches the 200 Hz IMU measurements into groups of 10 (one per camera frame).

```bash
# Install dependencies (can reuse the levio conda environment from the Python model)
conda activate levio

# Convert a rosbag
cd levio_gap9_project
python scripts/prepare_euroc_data.py <path-to-euroc>/MH_01_easy.bag --output_dir sensor_data/demo
```

This generates the `images/`, `imu/` directories and `start_stop_indices.h` as described above.

## Build

```bash
# Source the GAP SDK environment (if not already done)
source <path-to-gap-sdk>/configs/gap9_v2.sh

# Configure
cd levio_gap9_project
cmake -B build
```

## Run

```bash
cmake --build build --target run
```

This executes the VIO pipeline on the demo sequence using the GVSOC simulator bundled with the GAP SDK. The pipeline processes 160x120 grayscale frames at 20 Hz with IMU data at 200 Hz.

### Visualizing the Trajectory

Capture the console output to a log file, then use the visualization script to render a top-down trajectory:

```bash
# Run the pipeline and save the log
cmake --build build --target run 2>&1 | tee log/run.log

# Visualize (uses the levio conda environment from the Python model)
conda activate levio
python scripts/visualize_log.py log/run.log
```

This parses keyframe and regular frame poses from the log and displays them in a live OpenCV window. Keyframes are drawn in red, regular frames in blue. The trajectory is saved as `log/run_trajectory.png`.

## Configuration

All tunable parameters are in [definitions/config.h](definitions/config.h):

### Camera

| Parameter | Default | Description |
|-----------|---------|-------------|
| `IMG_W` / `IMG_H` | 160 / 120 | Input image resolution |
| `FX`, `FY` | 114.66, 114.32 | Focal lengths (pixels) |
| `CX`, `CY` | 77.80, 62.09 | Principal point (pixels) |
| `FRAME_RATE` | 20 Hz | Camera frame rate |

### Feature Detection (ORB)

| Parameter | Default | Description |
|-----------|---------|-------------|
| `FAST_THRESHOLD` | 20 | FAST corner intensity difference |
| `HARRIS_THRESHOLD` | 5000 | Harris corner response threshold |
| `PATCH_SIZE` | 8 | Image patch size for detection |
| `FEATURES_PER_PATCH` | 4 | Max features retained per patch |
| `HAMMING_THRESHOLD` | 35 | Max Hamming distance for descriptor match |

### RANSAC

| Parameter | Default | Description |
|-----------|---------|-------------|
| `RANSAC_ITERATIONS_EPNP` | 64 | EPnP RANSAC iterations |
| `RANSAC_ITERATIONS_ESSENTIAL` | 640 | Essential matrix RANSAC iterations |
| `MIN_MATCHES_EPNP` | 25 | Minimum matches to attempt EPnP |
| `REPROJECTION_ERROR_THRESHOLD` | 2.0 | Inlier threshold (pixels squared) |

### Pose Graph

| Parameter | Default | Description |
|-----------|---------|-------------|
| `MAX_KEYFRAMES` | 8 | Sliding window size |
| `MAX_WORLD_FEATURES` | 1000 | Maximum 3D landmarks |
| `PARALLAX_THRESHOLD` | 0.05 rad | Keyframe insertion threshold |
| `MAX_WORLD_POINT_AGE` | 15 | Frames before landmark is discarded |

### Execution

| Parameter | Default | Description |
|-----------|---------|-------------|
| `CLUSTER_FREQ` | 370 MHz | Cluster operating frequency |
| `L1_WORK_MEMORY` | 110 KB | L1 scratchpad allocation |
| `SINGLE_CORE` | 1 | Set to 0 for 8-core parallel execution |

## Architecture

### Memory Hierarchy

```
┌─────────────────────────────────────────┐
│  L2 Memory (off-cluster)                │
│  ├── Pose graph (keyframe poses,        │
│  │   world points, descriptors)         │
│  ├── IMU factors & observations         │
│  └── Image buffers (DMA source)         │
├─────────────────────────────────────────┤
│  L1 Scratchpad (110 KB, shared)         │
│  ├── Current frame image (14.4 KB)      │
│  ├── Keypoint & descriptor buffers      │
│  ├── Matrix computation workspace       │
│  └── Dynamically managed via work_memory│
└─────────────────────────────────────────┘
```

Data is transferred between L2 and L1 via DMA. The `work_memory` module provides a stack-style allocator over the L1 scratchpad with usage tracking.

### Multi-Core Parallelism

When `SINGLE_CORE=0`, the 8-core cluster is used for:

- **Feature detection** — Image is divided into patches; each core detects features in its assigned patches and maintains a per-patch min-heap ranked by Harris score.
- **Descriptor matching** — World descriptors are partitioned across cores; each core computes Hamming distances for its subset.
- **Triangulation** — Point triangulation is distributed across cores.

### Per-Frame Processing

```
1. DMA: Load image from L2 → L1
2. ORB detection: FAST corners → Harris scoring → BRIEF descriptors
3. Matching & pose estimation:
   a. If map has landmarks: EPnP RANSAC (absolute pose)
   b. Else: Essential matrix RANSAC + IMU-based scale
4. Triangulate new landmarks from matched features
5. Keyframe decision (parallax threshold)
6. If keyframe:
   a. IMU preintegration (trapezoidal, with bias Jacobians)
   b. Windowed pose graph optimization (Gauss-Newton)
   c. Gravity refinement
```
